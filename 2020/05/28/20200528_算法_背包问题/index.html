背包问题中的降维优化

若转移使用的是上一层（之前的）的状态，则遍历体积时应从大到小；用的是本层的状态，则从小到大遍历体积。

01背包

有 $N$ 个物品和容量为 $V$ 的背包，每个物品有体积 $vi$ 和价值 $wi$ 两个属性，每个物品最多只能用一次。在这些物品中选择物品使得总体积小于 $V$，且总价值尽可能高。

输入格式

第一行两个整数 $N$，$V$，用空格隔开，分别表示物品数量和背包容积。

接下来有 $N$ 行，每行两个整数 $v_i$，$w_i$，用空格隔开，分别表示第 $i$ 件物品的体积和价值。

数据范围

$0< N,V ≤ 1000$
$0< v_i, w_i ≤ 1000$

优化版

01背包问题可以进行优化。二维数组可以优化为一维滚动数组（只优化数组，不删除外层循环）。

可以发现：

  1. $f[i][j]$的两个状态都来自于上一层$[i - 1]$或同一层$[i]$；
  2. $f[i - 1][j]$和$f[i - 1][j - v[i]]$中的 $体积$ 项都小于或等于 $j$。

在朴素版代码中稍作修改，即可得到优化版。

注意，$f[j]$的状态转移来自于$max(f[j], f[j - V[i]] + W[i])$，由于 $j \geq j - V[i]$，若体积$j$仍从小到大枚举，则会先计算较小的$j$，则转移方程中的$f[j - V[i]]$将不再来自于$i - 1$而变成来自于$i$（因为即一维数组中来自于上一层的数会被本层新计算出的值覆盖，这样的循环顺序会导致串联），需要将体积从大到小枚举，避免来自于$i - 1$层的值在转移前被$i``层的值覆盖。
